# خطة بناء باك اند (Backend) بديل

## 1. تحليل الوضع الحالي

التطبيق يستخدم حالياً رابط `https://sonic-zdi0.onrender.com/api` ولكنك لا تملك صلاحية الوصول للوحة التحكم الخاصة به.
لذلك، الحل الوحيد هو **بناء باك اند جديد** بنفس المواصفات تماماً واستضافته على حسابك الخاص.

## 2. المتطلبات الفنية (ماذا يحتاج التطبيق؟)

بعد فحص الكود، التطبيق يحتاج فقط إلى الخدمات التالية من الباك اند:

### أ. المصادقة (Authentication)

- `POST /login`: تسجيل الدخول (إيميل + كلمة مرور).
- `POST /register`: تسجيل جديد (اسم + إيميل + كلمة مرور).
- `GET /profile`: جلب بيانات المستخدم (يتطلب Token).
- `POST /update-profile`: تحديث البيانات (اسم، إيميل، صورة).
- `POST /logout`: تسجيل الخروج.

### ب. المنتجات (Products)

- `GET /products`: جلب المنتجات (مع دعم الفلترة بالسعر، البحث، والصفحات).
- `GET /products/:id`: جلب تفاصيل منتج واحد.
- `GET /categories`: جلب قائمة التصنيفات.
- `GET /products/top-rated`: المنتجات الأعلى تقييماً.
- `GET /products/new`: المنتجات الجديدة.

### ج. ميزات غير مفعلة حالياً (موجودة في الكود لكن لا تعمل)

- `Orders`: سجل الطلبات (الصفحة موجودة لكنها تعرض بيانات وهمية).
- `Favorites`: المفضلة (غير مستخدمة).

## 3. خطة العمل المقترحة

### الخطوة الأولى: بناء الكود (دوري أنا)

سأقوم بإنشاء مجلد جديد اسمه `backend` داخل المشروع يحتوي على:

1.  **Node.js + Express**: لبناء السيرفر.
2.  **MongoDB**: قاعدة بيانات لتخزين المستخدمين والمنتجات.
3.  **JWT**: لتأمين الاتصال (Token).
4.  **Multer**: لرفع الصور (مثل صورة البروفايل).

### الخطوة الثانية: الاستضافة (دورك أنت)

بعد أن أسلمك الكود، ستحتاج لرفعه على استضافة. أرشح لك **Render** (نفس الاستضافة الحالية) لأنها سهلة ومجانية للتجربة.
سأعطيك ملف `README` يشرح خطوات الرفع بالتفصيل.

## 4. هل نبدأ؟

إذا وافقت، سأبدأ فوراً بكتابة كود الباك اند كاملاً في مجلد `backend`.
